Algoritmos

Centrality algorithms - Similarity algorithms

CONCEPTO

Similarity algorithms (Algoritmos de Similitud)

Los algoritmos de similitud calculan la similitud de pares de nodos utilizando diferentes métricas basadas en vectores.

  • Calidad de Produccion
  • Node Similarity

Centrality algorithms (Algoritmos de Centralidad)

Los algoritmos de centralidad se utilizan para determinar la importancia de distintos nodos en una red.

  • Calidad de Produccion
  • Betweenness Centrality

Problema:

Una persona o cliente desea conocer a los usuarios que hayan realizado un publicacion similar a otros usuarios y al mismo tiempo observar los nodos con mayor importancia

Solucion

Para conocer los nodos que son similares dentro de una base de datos, usaremos el algoritmo de similitud para conocer los nodos resultantes al problema, dentro de la base de graficos (Neo4j) instalaremos la libreria GDS, Neo4j para (Graph Data Science) incorpora el poder predictivo de las relaciones y las estructuras de red en los datos existentes para responder preguntas previamente intratables y aumentar la precisión de la predicción.

Tambien para descubrir los nodos centralizado usaremos el algoritmo centralizado, el cual nos permite ver el nodo con mayor importancia de distintos nodos.

Creamos un Grafico dentro de la base de datos, esto permite que varios algoritmos utilicen el mismo gráfico sin tener que volver a crearlo en cada ejecución del algoritmo. 


                            CALL gds.graph.create(
                                'myGraph',
                                ['Presidente', 'Tweets','Usuario'],
                                {
                                    LIKES: {
                                        type: 'TWEET_PRESIDENTE',
                                        properties: {
                                            strength: {
                                                property: 'score',
                                                defaultValue: 1.0
                                            }
                                        }
                                    }
                                }
                            );

Ejecutamos la siguiente sentecia cypher para visializar el resultado de los nodos de similitud 


    CALL gds.nodeSimilarity.stream('myGraph')
    YIELD node1, node2, similarity
    RETURN gds.util.asNode(node1).name AS Usuario1, gds.util.asNode(node2).name AS Usuario2, similarity
    ORDER BY similarity DESCENDING, Usuario1, Usuario2

Ahora haciendo uso del algoritmo centralizado para ver los nodos mas importantes. 


    CALL gds.betweenness.stream('myGraph')
    YIELD nodeId, score
    RETURN gds.util.asNode(nodeId).name AS name, score
    ORDER BY name ASC
